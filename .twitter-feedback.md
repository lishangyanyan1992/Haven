## Proven Tweet Patterns (from research on top build-in-public creators)

### Pattern 1: The Contrarian Hook
Bold statement that challenges conventional wisdom + brief evidence.
Template: "I [do unusual thing]. [Brief reason]. I think [provocative opinion]?"
Example: "I don't write code anymore. I haven't written code in 6 months. I think everyone is like this no?"

### Pattern 2: The Specific Numbers Flex
Exact revenue/users/lines of code + unexpected approach. Numbers make claims credible and shareable.
Template: "[Product] makes [exact $/users] using [unexpected stack/approach]. That's [X]x [comparison]."
Example: "Photo AI is 14,000 lines of raw PHP making $61,808/mo. 1,872 paying customers."

### Pattern 3: The Numbered Insight
Identify a problem everyone feels → break it down into numbered points. Scannable, authoritative, retweetable.
Template: "The real challenge with [topic] isn't [obvious thing]. It's: 1) [point], 2) [point], 3) [point]"

### Pattern 4: The David vs. Goliath Story
Solo/indie vs. big/funded → outcome. Underdog narrative is universally compelling.
Template: "I [competed against] a [big entity] doing [same thing]. Here's what happened."

### Pattern 5: The Rapid Build/Ship
Built [thing] in [short time] + result. Time compression creates awe.
Template: "I built [product] in [short time]. [Result/metric]. Here's how →"

### Pattern 6: The Field Report
Second-hand observation format. Less self-promotional, more credible.
Template: "Someone I know [did thing] and observed: '[surprising quote].' This is [takeaway]."

### Pattern 7: The Extreme Cost/Value Comparison
What it would cost [normal way] vs. what I actually pay [my way]. Math is undeniable.
Template: "[Normal approach] would cost $[big number]. I pay $[small number]. That's [X]x less."

## Writing Rules (from analyzing viral build-in-public tweets)

1. Lead with the most surprising number or claim — first line = title
2. Use specific numbers everywhere (not "growing" → "3 signups today")
3. Create contrast/tension (unfashionable method + great result, solo + vs funded, fast + profitable)
4. Write in short, punchy lines — each sentence gets its own line break. No paragraphs.
5. Have a strong opinion — neutral tweets from unknown accounts get zero traction
6. End with engagement bait naturally (rhetorical questions, "what do you think?")
7. Share the "how" not just the "what" — promise of learning drives saves/retweets
8. Show your stack, not just your results — tech details create debate
9. Be vulnerable about mistakes and failures — struggles make it compelling
10. Use emoji as visual bullet points (✨💳) — they break up text and draw the eye
11. Include screenshots when possible — tweets with images outperform text-only
12. Make it teachable — imply "here's how you can do X too"

## Common Mistakes to Avoid

- Vague humble-bragging ("So grateful to announce...") → instead be direct ("I made $X")
- Only posting wins → share failures, pivots, doubts
- No numbers → "growing nicely" is useless, "3 signups, up from 0" is a story
- Broadcasting instead of engaging → reply to comments, ask questions
- Over-engineered threads → keep it short, 3-5 lines max
- No opinion → pure factual updates get ignored
- Only talking about features → "Added dark mode" is boring, "40% of users forced dark mode via browser hack, so I built it properly" is interesting
- Trying to look bigger than you are → "0 followers, day 1" is MORE compelling than pretending you're established
